## 🎓 Kavramlar (Şeyma için not — yazılım öğrenmek)

- **Peer-via-direct-invoke** = Sales Director'ın Reklam Uzmanı'na soru
  sorma yöntemi. Tool içinde başka bir agent'ı çalıştırıyoruz (Runner.run).
  Avantajı: senkron cevap, agent zinciri görünür. Dezavantajı: bir LLM
  içinde başka LLM çağrısı = ekstra latency.
- **Deterministik helper** = LLM yerine pure Python kodu. Aynı input =
  aynı output. Test'lerle %100 garanti. `brand_aware_prompt` builder buna
  örnek — LLM'in iş yükünü azalttı.
- **Defense-in-depth** = aynı kuralı 2 yerde sağla. Sales Director'ın
  post atmaması hem instructions'da (LLM uyarısı) hem tool listesinde
  (`post_on_instagram` yok) garanti ediliyor. LLM unutsa bile tool yok =
  yapamaz.
- **Knowledge layer paylaşımı** = aynı 5 tool hem Sales hem Marketing'de.
  DRY (Don't Repeat Yourself) prensibi. Brand bilgisi değişirse tek yerde
  güncellenir.
- **ADR-001 sınırı** = "NocoDB = CRM, Firestore = medya/ops" kuralı. Bu
  yüzden içerik takvimini Firestore'da tuttuk, NocoDB tablosu açmadık.
